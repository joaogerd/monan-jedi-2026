# 1. Contexto Institucional

A implementação estruturada do sistema JEDI + MONAN exige ambiente computacional
reproduzível, congelado e institucionalmente validado.

Experiências anteriores demonstraram que ausência de controle de dependências,
configuração manual de módulos e ambientes paralelos não documentados
comprometem a continuidade técnica e dificultam a manutenção do sistema.

Este marco estabelece a base infraestrutural sobre a qual todas as fases
científicas subsequentes serão executadas.

Sem ambiente consolidado, não há validação científica robusta.

---

# 2. Objetivo Técnico

Estabelecer ambiente computacional oficial, versionado e reprodutível
para compilação e execução do MPAS-JEDI no cluster institucional (JACI).

O ambiente deve permitir:

- Build determinístico
- Execução via PBS
- Reprodutibilidade cruzada entre usuários
- Registro completo de dependências

---

# 3. Escopo

## Inclui

- Definição oficial do compilador
- Definição oficial da implementação MPI
- Configuração completa via Spack
- Geração de `spack.yaml`
- Geração de `spack.lock`
- Build completo do mpas-bundle
- Execução teste via PBS
- Validação cruzada por pelo menos dois membros

## Não Inclui

- Execução de experimentos científicos
- Ajustes físicos no modelo
- Integração MONAN
- Configuração de 3DVar
- Avaliação de métricas científicas

---

# 4. Dependências Formais

- Acesso funcional ao cluster JACI
- Permissões de uso do Spack
- Disponibilidade do repositório mpas-bundle
- Disponibilidade do compilador institucional

Não depende de marcos anteriores.
